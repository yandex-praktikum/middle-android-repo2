### Задание 1

Для начала потребуется заменить использование `LiveData` на `StateFlow` для оптимизации работы с данными.

Сейчас `LiveData` находится внутри `ChatViewModel`. Для перевода на `StateFlow` требуются следующие шаги:

1. Создать `StateFlow<List<String>>` в `ChatViewModel` и удалить использование `LiveData`.
2. Переписать метод `sendMessage(String)`, только теперь с использованием ранее созданного `Flow`.
3. Внутри MainActivity заменить `observeAsState` на `collectAsState` для `messages`.

### Задание 2

В этом задании вам необходимо написать логику обработки ошибок «сети». Конечно, настоящих запросов в сеть в проекте нет и ошибки будут эмулироваться  в `ChatApi`.

1. Раскомментируйте строчку со случайным бросанием ошибки в `ChatApi`. Судя по коду, запрос за ответным сообщением будет возвращаться с ошибкой с заметной регулярностью.

2. Добавьте обработчик в метод `getReplyMessage()` из класса `ChatRepository`, который будет повторять запрос несколько раз с экспоненциальным увеличением тайм-аута. Для этого пригодится функция `retryWhen`, в которой вы должны написать условие на проверку ошибки и в этом случае вызывать `delay` с заданным временем задержки, после которого увеличивать время задержки.

3. Чтобы быть уверенным в надёжности данной логики, напишите юнит-тесты, которые будут проверять обработку ошибки и успешное получение.

   В файле `ChatRepositoryTest` есть два метода — один для проверки успешного ответа, другой для проверки ретрая. Код для проверки ретрая уже написан — так вы можете проверить, что логика, реализованная вами в репозитории, работает правильно. Напишите по аналогии тест для проверки успешного ответа без ретрая.

### Задание 3

Для удобства пользователя после появления первого сообщения на экране чата нужно открыть клавиатуру и поместить курсор в поле ввода:

1. В `ChatViewModel` добавьте состояние `shouldShowKeyboard` которое будет становиться `true` после получения первого сообщения.
2. По `shouldShowKeyboard` запрашивайте фокус для поля ввода. Для этого нужно воспользоваться `FocusRequester`:

```kotlin
@Composable
fun ChatScreen() {
    val focusRequester = remember { FocusRequester() }
	
	BasicTextField(
		modifier = Modifier
			.focusRequester(focusRequester)
	)

}
```

И по получении первого сообщения, вызовите один раз `focusRequester.requestFocus()`

### Задание 4

В предыдущем задании вы добавили состояние `shouldShowKeyboard`. Такой код будет работать, однако, наличие у `ViewModel` нескольких независимых состояний может принести проблемы в будущем — они могут рассинхронизироваться и из-за этого в приложении появятся баги.

Чтобы этого избежать, отрефакторите своё решение:

1. Создайте новую сущность `ChatState`, которая будет полностью описывать состояние экрана.
2. В `ChatViewModel` замените свойства `messages` и `shouldShowKeyboard` на `chatState`.

### Задание 6

&[В данном задании вы напишете юнит-тест, который проверит потокобезопасность нашей реализации.](375092)

Внутри метода `testReceiveMessage_concurrentMessages` напишите тест по аналогии с тестом из задания 2, но который будет одновременно отправлять 100 сообщений из списка `messagesToSend`.

{% cut "Подсказка" %}

Вам нужно запустить несколько корутин одновременно. Как это сделать?

Создать `coroutineScope` и для каждого сообщения сделать fire-and-forget корутину. Необходимо дождаться окончания выполениния всех корутин. Здесь пригодится использование extension-функции `Collection<Job>.joinAll` для параллельного запуска сразу 100 добавлений в список сообщений.

Далее мы должны проверить количество сообщений и их содержание.
